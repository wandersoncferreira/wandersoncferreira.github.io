Functional Programming

February 25, 2020
functional

Youtube Link

Speaker

Erik Meijer, computer scientist and software developer

Thoughts

Very good explanation about FP. It uses Haskell a lot as example but all the concepts are easy to grasp. I made parallels with Clojure while reading his lesson.

Quotes

The concept of honest and disonest programming languages is about tricking you to think the functions and method behaves in specific way that you can’t predict.

A lambda expression is a delegate.

class Delegate<T,R> {
   R invoke(T t)
   }

A class is a superset of lambda expressions in .NET and other languages. Lambda expressions is not about FP or Purity, they are all about capturing closures.

Recommendations

  1. FP definition is simple: Programming with mathematical functions!
  2. Everytime someone wants to make effects you have to pass a global variable around. You have to pass the whole world around.
  3. Lambda expressions can provide you with closures and anonymous fns.
