Condition System in an Exceptional Language

May 31, 2020
clojure

Youtube

Speaker

Cris Houser

Thoughts

Very nice presentation about how to handle errors.

Quotes

“Vast opportunities that we software developers have to make models that prove and expand human knowledge, but we’are too busy plumbing”

Recommendations

  1. Consider alternatives to throwing exceptions
  2. Try out the built-in condition system, knowing this technique can take you as far as you’re likely to need
  3. Use an existing condition system library where it makes sense.

Books

  1. Software Reliability, Principles & Practices - by Glenford
  2. Practical Common Lisp - by Peter Seibel
  3. The adventure of a blunder
